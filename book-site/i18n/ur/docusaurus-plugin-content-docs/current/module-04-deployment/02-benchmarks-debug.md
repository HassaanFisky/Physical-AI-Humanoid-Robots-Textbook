# بینچ مارکس اور ڈیبگنگ

![روبوٹک ڈیٹا کا تجزیہ](https://images.unsplash.com/photo-1551288049-bb8c8334a194?w=1200&h=600&fit=crop)

> **خلاصہ:** اگر آپ اسے ناپ نہیں سکتے، تو آپ اسے بہتر نہیں بنا سکتے۔ روبوٹکس میں کارکردگی کو جانچنا اور غلطیوں کو دور کرنا ایک فن ہے۔

## تعارف: پیمائش کی اہمیت

روبوٹکس سسٹم بہت پیچیدہ ہوتے ہیں۔ جب کوئی چیز غلط ہوتی ہے، تو یہ جاننا کہ مسئلہ کہاں ہے (سافٹ ویئر، ہارڈویئر، یا سینسر) سب سے بڑا چیلنج ہوتا ہے۔

---

## 1. کارکردگی کے پیمانے (KPIs)

روبوٹ کی کارکردگی جانچنے کے لیے ہم درج ذیل چیزیں دیکھتے ہیں:
- **کنٹرول فریکوئنسی (Control Frequency)**: سسٹم کتنی جلدی فیصلے کر رہا ہے (مثلاً 1000Hz)۔
- **تاخیر (Latency)**: سینسر ڈیٹا آنے اور حرکت شروع ہونے کے درمیان کتنا وقت لگتا ہے۔
- **درستگی (Accuracy)**: روبوٹ اپنے ہدف سے کتنا دور ہے۔
- **توانائی کی کھپت**: بیٹری کتنی جلدی ختم ہو رہی ہے۔

---

## 2. ڈیبگنگ کے اوزار (Tools)

### Foxglove اور Rviz
یہ اوزار روبوٹ کے دماغ کے اندر دیکھنے میں مدد دیتے ہیں۔ آپ سینسر ڈیٹا، 3D نقشے اور کیمرہ فیڈ کو لائیو دیکھ سکتے ہیں۔

### لاگنگ (Logging)
روبوٹ کے ہر فیصلے کو فائل میں محفوظ کرنا (ROS Bags) تاکہ بعد میں اس کا تجزیہ کیا جا سکے۔

---

## 3. عام مسائل اور حل

| مسئلہ | ممکنہ وجہ | حل |
| :--- | :--- | :--- |
| **توازن کا بگڑنا** | سینسر میں شور (Noise) | فلٹر بہتر کریں |
| **حرکت میں جھٹکے** | کنٹرول لوپ میں تاخیر | کوڈ کو بہتر (Optimize) کریں |
| **رستے سے بھٹکنا** | غلط اسٹیٹ ایسٹیمیشن | سینسر فیوژن چیک کریں |

---

## کلیدی معلومات

:::note خلاصہ

1. **پیمائش** بہتر کارکردگی کے لیے لازمی ہے۔
2. **بر وقت ڈیٹا** (Real-time data) کی نگرانی کریں۔
3. **Foxglove** جیسے اوزار ڈیبگنگ کو آسان بناتے ہیں۔
4. **ٹیلرڈ ٹیسٹنگ** (Simulation + Real) استعمال کریں۔
   :::
